# 1. システム概要

## 1.1 プロジェクトの目的

本プロジェクトは、民泊施設におけるゲストの滞在体験を革新し、同時に施設運営の効率化と新たな収益機会の創出を目指すBYOD（Bring Your Own Device）型コンシェルジュシステムを開発します。ゲスト自身のスマートフォンを通じて、チェックイン・アウト、アメニティリクエスト、宿泊税支払い、観光案内といった多様なサービスを、AIが媒介する統一されたインターフェースで提供します。

## 1.2 主要機能の概要

### AIコンシェルジュ（必須機能）
*   **役割:** 各サービス機能への「総合的なゲートウェイ」として機能します。ゲストの自然言語による問い合わせに対し、AIが適切な情報提供、提案、または他の機能への誘導を行います。
*   **特徴:** ユーザーの意図を解釈し、パーソナライズされた応答や、地図表示、リンク、クイックリプライなどの構造化された提案を返します。

### その他の連携機能 (AIコンシェルジュ経由でアクセス)
*   オンラインチェックイン・アウト
*   アメニティリクエスト
*   宿泊税オンライン決済
*   観光・交通情報案内
*   施設からのお知らせ・緊急連絡

## 1.3 システムアーキテクチャ

システムは、フロントエンド、バックエンド、および外部AIサービス（Gemini API）から構成されるクライアント・サーバーアーキテクチャを採用します。

```mermaid
graph LR
    A[ゲストのスマートフォン] -- HTTPS --> B[フロントエンド: React App]
    B -- REST API (HTTPS) --> C[バックエンド: Node.js (Express)]
    C -- Gemini API (HTTPS) --> D[Google Gemini AI]
    C -- Data (JSON/DB) --> E[データストア: 推薦リスト, ゲスト情報など]
```

**説明:**
*   **ゲストのスマートフォン:** ゲスト自身のデバイス。ブラウザを通じてWebアプリケーションにアクセスします。
*   **フロントエンド (React App):** ゲストが直接操作するUIを提供します。チャット画面、各機能の表示・入力フォームなどを担当します。バックエンドAPIとの通信を行います。
*   **バックエンド (Node.js/Express):** フロントエンドからのリクエストを受け取り、Gemini APIとの連携、データストアからの情報取得、ビジネスロジックの実行、そしてフロントエンドへのレスポンス返却を行います。
*   **Google Gemini AI:** バックエンドからの指示（プロンプト）に基づき、ゲストへの応答や提案を生成します。
*   **データストア:** 施設が推薦するスポットリスト、ゲストの好み（将来的に）、施設情報などを格納します。初期段階ではJSONファイルで実現し、将来的にデータベースへ移行する可能性があります。
